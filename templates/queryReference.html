Query Reference

Query Types

SourceControl's query language allows for 4 different types of queries

find OBJECT where CONDITIONS

Returns objects matching a given set of conditions

get DATA from OBJECT where CONDITIONS

Returns the specified data type from the objects found using the given query parameters.

count OBJECT where CONDITIONS

Just like 'find' queries, but simply returns a count of the data points gathered.


Condition Syntax

Conditions take the form

DATA OP VALUE

where DATA is a type is a data point associated with the OBJECT. OP is one of '>', '<', '=', '!=', 'in', 'not in', 'contains'. Value is simply the value to compare against. For strings, the value must contained in double quotes.


Object Types

Used in place of OBJECT in queries.

users

Used to query on committers of a repository. Data point used for 'find' queries is num_commits.

DATA points

num_commits - Number of commits the user has made

additions - Number of lines added

deletions - Number of lines deletion

num_break_build - Number of builds broken by this user

num_fix_build - Number of builds fixed by this user

num_build - Number of builds by this user


commits

Find commits on the repository. Data point used for 'find' queries is commit_time.

DATA points

commit_time - Time of commit. Use string with date for condition.

num_patches - Number of changes made within a commit. Useful as a metric for commit size.

build_break_status - 1 if build was broken with this commit. 0 otherwise.


branches

Find branches on the repository. Data point used for 'find' queries is num_commits.

DATA points

num_commits - Number of commits on this branch.

num_files - Number of files on this branch.


files

Find files within repository. Data point used for 'find' queries is num_edit.

DATA points

additions - Number of additions made to this file over all time.

deletions - Number of deletions made to this file over all time.

num_edit - Number of edits made to this file over all time.
